# Code Review - 代码审查技能

## 概述

这是一个代码审查技能，帮助 AI 助手进行系统化的代码审查。

## 使用场景

- 提交前代码审查
- Pull Request 审查
- 代码质量检查

## 审查清单

### 代码质量
- [ ] 代码是否清晰易读
- [ ] 命名是否语义化
- [ ] 是否有重复代码
- [ ] 函数是否过长（建议 < 50 行）

### 安全检查
- [ ] 是否有硬编码密钥
- [ ] SQL 注入风险
- [ ] XSS 漏洞
- [ ] 敏感数据处理

### 性能考虑
- [ ] 是否有不必要的循环
- [ ] 数据库查询是否优化
- [ ] 是否有内存泄漏风险

### 最佳实践
- [ ] 错误处理是否完善
- [ ] 日志记录是否合理
- [ ] 是否遵循项目规范

## 审查输出模板

```markdown
## 代码审查报告

**文件**: [文件名]
**审查时间**: [日期]

### 发现的问题

| 级别 | 位置 | 描述 | 建议 |
|------|------|------|------|
| 🔴 高 | L10 | 硬编码 API Key | 移至环境变量 |
| 🟡 中 | L25 | 未处理异常 | 添加 try-catch |
| 🟢 低 | L40 | 命名不清晰 | 改用语义化命名 |

### 优点
- 

### 建议改进
- 
```

## 使用方式

```
请对这段代码进行审查，使用 code-review 技能
```
